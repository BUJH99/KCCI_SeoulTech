케이스별 ideal 상대오차

가속기 근사 경로 이후의 케이스별 ideal softmax 평균/최대 상대오차입니다.

의미

스위트 수준의 근사 품질 차트입니다. 각 케이스 출력이 수학적 ideal softmax에서 얼마나 벗어나는지 보여줍니다.

근거

파란/주황 막대는 케이스별 ideal_total 통계에서 오며, input_frames.jsonl과 actual_frames.jsonl을 사용해 DUT FP32 출력을 수치 확률로 복원해 계산합니다.

사용 이유

RTL 골든과 exact match된다는 이유만으로 안심하는 것을 막고, ideal 알고리즘 대비 실제 근사 간극을 드러내기 위해 사용합니다.

RTL 대 ideal 지표 대비

RTL mismatch가 0%여도 ideal softmax 오차가 0이 아닐 수 있는 이유를 보여줍니다.

의미

구현 충실도와 알고리즘 충실도를 분리해서 보여주는 차트입니다. 두 숫자는 서로 다른 질문에 답하므로 값이 다를 수 있습니다.

근거

RTL mismatch는 RTL 충실 Python 모델과의 scoreboard exact compare에서 오고, ideal 평균/최대 상대오차는 동일 출력에 대한 ideal-softmax 후처리 분석에서 옵니다.

사용 이유

RTL mismatch 0%를 approximation error 0%로 잘못 해석하는 것을 막기 위해 사용합니다. 이 오해는 이 프로젝트에서 실제로 문제였습니다.

ideal 분석 대상 여부

비유한 입력을 가진 프레임은 ideal softmax 분석에서 제외되며 따로 집계됩니다.

의미

ideal softmax 지표의 분모가 되는 모집단 크기를 보여줍니다. 모든 프레임이 수학적으로 이 비교에 적합한 것은 아닙니다.

근거

대상/제외 개수는 compute_ideal_metrics에서 계산하며, 비유한 입력 또는 invalid ideal-softmax 재구성 프레임을 명시적으로 제외합니다.

사용 이유

이 차트가 없으면 실제로 몇 프레임이 평균 계산에 들어갔는지 모른 채 ideal 오차 평균만 과신할 수 있습니다.

케이스별 확률합 residual

RTL mismatch가 0이고 커버리지가 높아도 정규화 drift를 눈에 보이게 만듭니다.

의미

확률 질량 보존 관점에서 정규화 정확도를 케이스 단위 신호로 압축한 차트입니다.

근거

각 막대는 ideal_by_frame의 sum_residual 값을 케이스 평균/최대 절대 residual로 집계한 결과입니다.

사용 이유

softmax 가속기는 RTL 모델과 일치해도 확률 질량이 새거나 과도하게 커질 수 있으므로, 그 실패 모드를 스위트 레벨에서 보이게 하기 위해 사용합니다.

스위트 시나리오 coverage snapshot

통합 보고서 첫 페이지에서 coverage가 낮은 시나리오들을 바로 확인할 수 있게 정리한 요약 차트입니다.

의미

근거

사용 이유

앞페이지에서 바로 약한 시나리오를 찾고, 케이스 페이지를 열기 전에 추가 자극이 필요한 구간을 빠르게 식별하기 위해 사용합니다.

이 차트는 시나리오가 검증 공간을 넓게 덮는지와 동시에 근사 파이프라인에서 수치적으로 어려운지도 함께 보여줍니다.

프레임 오차 추이

막대는 ideal softmax 대비 평균 절대오차이고, 빨간 선은 프레임별 최대 상대오차입니다.

의미

프레임 간 수치 안정성 보기입니다. 근사 품질이 일정한지, 특정 프레임에서만 급격히 나빠지는지를 보여줍니다.

근거

모든 막대와 선 포인트는 ideal_by_frame에서 가져옵니다. 막대는 한 프레임의 avg_abs_err, 선은 같은 프레임의 max_rel_err_pct 입니다.

사용 이유

케이스 전체는 RTL exact compare를 통과해도 일부 프레임만 약할 수 있는데, 이 차트가 그런 outlier를 즉시 드러냅니다.

의미

정규화 품질을 직접 측정하는 차트입니다. 막대가 0에 가까울수록 출력 확률이 유효한 softmax 분포에 가깝습니다.

근거

사용 이유

상대오차 분포

프레임 최대 상대오차의 분포를 통해 근사 오차가 좁게 모이는지, 긴 꼬리를 갖는지 보여줍니다.

의미

최악값 하나만이 아니라 프레임 전체에서 근사 오차가 어떤 형태로 퍼져 있는지를 보여주는 히스토그램입니다.

근거

히스토그램 샘플 하나는 ideal 대상 프레임 하나의 max_rel_err_pct 값이며, ideal_by_frame에서 가져옵니다.

사용 이유

최악 프레임 오버레이

의미

사용 이유
